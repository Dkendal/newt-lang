# `dbg!` v2 (Span Watchpoints) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert `dbg!` from definition-site/static reporting to evaluation-site semantics — fire with live bindings when the engine demands the marked node, silent on dead branches — plus a `--trace-eval` engine-tracing flag.

**Architecture:** Spec: `docs/superpowers/specs/2026-07-05-dbg-macro-v2-watchpoints-design.md`. The erasure pass stays; it now emits a watch table of spans instead of printing. A shared `DbgSink` (watch table + deduped event log) hangs off `TypeEnv`; read-only "observe" hooks at the engine's demand points record `(span, current node)`; the assert harness flushes the sink after each claim, normalizing and rendering each event. No AST node type changes; the evaluated program is byte-identical to the erased program.

**Tech Stack:** Rust, ariadne, cargo nextest, mise/tsgo conformance.

## Global Constraints

- **No observer effect:** hooks are read-only observations. They must never unwrap, rewrite, or change any return value or control flow. Feature on/off must produce identical assert outcomes for every program.
- **Firing semantics:** fires iff evaluation demands the watched node; dead conditional branches silent; indeterminate conditionals fire both branches; unused definitions/`let` bindings silent; dedupe on `(watch span, fingerprint(observed))` — once per distinct instantiation.
- Output format unchanged from v1: `render_debug` report per event, label `= <normalized type>`.
- jj + git colocated repo: `jj commit -m "<msg>" <paths>` scoped to changed files; the unrelated `examples/ts-toolbelt.nt` working-copy edit must NOT be committed. Never `git commit`.
- `cargo nextest run` for tests; `cargo fmt` before each commit; `mise run tc` for conformance.
- Implementers may adapt the plan's code sketches to the codebase's actual internals (lifetimes, field shapes), but the **Interfaces** blocks are contracts between tasks — keep names and signatures.

### Task 1: Watch table — `expand` returns watches instead of printing

**Files:**
- Modify: `src/ast/dbg_expr.rs` (delete the evaluate-&-report phase; add `DbgWatches`; rewrite tests)
- Modify: `src/main.rs` (adjust the `expand` call site)

**Interfaces:**
- Produces:
  ```rust
  /// A single watched site: the span evaluation must demand for it to fire.
  pub struct DbgWatch {
      pub span: Span,
      /// `Some(name)` when the watched node is a bare identifier (a probable
      /// type-parameter reference): substitution must observe its replacement.
      pub bare_ident: Option<String>,
  }
  pub struct DbgWatches(/* private */);
  impl DbgWatches {
      pub fn is_empty(&self) -> bool;
      pub fn contains(&self, span: Span) -> bool;          // exact (start,end) match
      pub fn bare_ident(&self, span: Span) -> Option<&str>;
      pub fn iter(&self) -> impl Iterator<Item = &DbgWatch>;
  }
  pub fn expand(program: &Ast, source: &str, source_name: &str) -> (Ast, DbgWatches)
  ```
- `expand` keeps: strip-before-simplify erasure (MacroCall replaced by its argument; UnitTest/Assert/Interface arms; arity panic with source-anchored report), pipeline-step recovery via `from_pipe` peeling — but each collected call now contributes **watches** (one per pipeline step; the step's span; `bare_ident` set when that step is a bare `Ast::Ident`) instead of printed reports. The old `TypeEnv`/normalize/render code in `expand` is deleted. Signature loses `color`/`out`/`io::Result` — it no longer writes.

- [ ] **Step 1: Rewrite the tests first.** In `src/ast/dbg_expr.rs` `mod tests`: keep every erasure-identity assertion (cleaned render == dbg-free render) exactly as they are; delete all assertions on printed output (`out.matches("Debug")` etc. — there is no output from this pass anymore); update the `run` helper to `let (cleaned, watches) = expand(&parse_newtype_program(src).unwrap(), src, "<test>"); (cleaned.simplify(), watches)`. Add watch-table assertions:

```rust

    let src = "type T as dbg!(1)";
    let (_, watches) = run(src);
    let at = src.find('1').unwrap();

}

#[test]

    let src = "type Id(T) as T\ntype B as 1 |> Id |> dbg!()";

}

#[test]

    let src = "type Get(A, K) as dbg!(K)";
    let (_, watches) = run(src);
    let at = src.rfind('K').unwrap();
    assert_eq!(watches.bare_ident(crate::ast::Span::new(at, at + 1)), Some("K"));

```

### Task 2: `DbgSink` on `TypeEnv` + the relation-entry hook + harness flush

**Files:**
- Modify: `src/ast/dbg_expr.rs` (add `DbgSink`, `DbgEvent`)
- Modify: `src/ast/type_env.rs` (`TypeEnv` carries `Option<Rc<DbgSink>>`; accessor)
- Modify: `src/ast/assignability.rs` (one hook at `is_assignable_to_ctx` entry)
- Modify: `src/test_harness.rs` (wiring + per-claim flush + render)
- Modify: `src/main.rs` (thread watches into the harness)

**Interfaces:**
- Produces (in `dbg_expr.rs`):
  ```rust
  pub struct DbgEvent { pub span: Span, pub observed: Ast }
  pub struct DbgSink { /* watches: DbgWatches, events: RefCell<Vec<DbgEvent>>, seen: RefCell<HashSet<(usize, usize, String)>> */ }
  impl DbgSink {
      pub fn new(watches: DbgWatches) -> Self;
      /// Read-only hook body: if `node.as_span()` is watched, record a clone,
      /// deduped on (span, type_env::fingerprint(node)).
      pub fn observe(&self, node: &Ast);
      /// For the substitution hook (Task 3).
      pub fn observe_replacement(&self, ident_span: Span, replacement: &Ast);
      /// Drain events recorded since the last call.
      pub fn drain(&self) -> Vec<DbgEvent>;
  }
  ```
- Produces (in `type_env.rs`): `TypeEnv::with_dbg(self, sink: Rc<DbgSink>) -> Self` (builder) and `pub(crate) fn dbg(&self) -> Option<&DbgSink>`.
- Produces (in `assignability.rs`), at the top of `is_assignable_to_ctx`, before any matching:
  ```rust
  if let Some(sink) = ctx.env().and_then(|env| env.dbg()) {
      sink.observe(self);
      sink.observe(other);
  }
  ```
- Produces (in `test_harness.rs`): `run` accepts the watches (extend `Config` or add a parameter — implementer's choice, but `main.rs` and all existing callers must compile; watches default to empty). When non-empty: build the env with the sink; after **each** claim's evaluation, `sink.drain()`, and for each event print to the harness's existing `out` writer: `render_debug(source_name, source, event.span, &format!("= {}", event.observed.normalize(&ctx).render_pretty_ts(80)), false)`. (Color stays off inside the harness; `main` already gets color-off harness output today — keep consistent.)

- [ ] **Step 1: Failing test** in `src/test_harness.rs` tests (add a helper `run_src_dbg(src) -> (Report, String)` that parses, expands, simplifies, and runs with the watches):

### Task 3: Remaining demand hooks — reducers + substitution (the motivating case)

**Files:**
- Modify: `src/ast/assignability.rs` (hooks in `reduce_conditional`, `reduce_access_leaf`/`index_type`, `eval_keyof`)
- Modify: `src/ast/type_env.rs` (substitution hook for bare-parameter watches)
- Test: `src/test_harness.rs` (or a new `tests/dbg_eval.rs` integration test if harness tests get crowded)

**Interfaces:**
- Consumes: `DbgSink::observe` / `observe_replacement`, `DbgWatches::bare_ident` (Task 1/2).
- Hook placement contract: each hook is one read-only `observe` call on the node **as the reducer receives it** (before resolution), at:
  - `reduce_conditional`: the check type (`lhs`) on entry;
  - the indexed-access reduction path: the object side and the key as received;
  - `eval_keyof`: the argument as received;
  - substitution (`distribute_or_substitute`'s replacement of a parameter `Ident` — thread `Option<&DbgSink>` from `TypeEnv::instantiate` down, defaulting `None` for all other callers): when the replaced `Ident`'s span is watched, `observe_replacement(ident_span, argument)` — the event's span is the *watch* span, its observed node the argument.

- [ ] **Step 1: Failing tests** — the motivating end-to-end (mirror of the ts-toolbelt shape) plus per-hook probes:

```rust
#[test]
fn dbg_fires_with_live_bindings_and_dead_branches_stay_silent() {
    let src = "type User as { id: number, name: string }\n\
        type At(A, K) as if K <: keyof A then dbg!(A[K]) else dbg!(never) end\n\
        unittest \"t\" do\n  assert At(User, 'id') <: number\nend";
    let (report, out) = run_src_dbg(src);
    assert_eq!((report.passed, report.failed), (1, 0), "{out}");
    // The taken branch fires with the substituted, demanded node…
    assert_eq!(out.matches("Debug").count(), 1, "{out}");
    // …and the value reflects the live binding (A[K] with A=User, K='id').
    assert!(out.contains("= number") || out.contains("User"), "{out}");
    // The dead else-branch (dbg!(never)) printed nothing:
    assert!(!out.contains("= never"), "{out}");
}

#[test]
fn bare_parameter_dbg_reports_the_argument() {
    let src = "type Probe(K) as dbg!(K)\n\
        unittest \"t\" do\n  assert Probe('id') <: string\nend";

    assert_eq!((report.passed, report.failed), (1, 0), "{out}");
    assert_eq!(out.matches("Debug").count(), 1, "{out}");

        unittest \"t\" do\n  assert Probe('a') <: string\n  assert Probe('b') <: string\nend";
    let (_, out) = run_src_dbg(src);
    assert_eq!(out.matches("Debug").count(), 2, "{out}");
}

#[test]
fn keyof_operand_dbg_fires() {
    let src = "type User as { id: number }\n\
        unittest \"t\" do\n  assert 'id' <: keyof dbg!(User)\nend";
```

### Task 4: `--trace-eval`

**Files:**
- Modify: `src/main.rs` (clap flag), `src/ast/type_env.rs` (instantiate trace), `src/ast/assignability.rs` (conditional-decision trace), `src/test_harness.rs` or `dbg_expr.rs` (wherever the trace sink naturally lives — implementer's judgment, but keep it on the same `DbgSink`/`TypeEnv` plumbing: e.g. a `trace: bool` on `DbgSink` that, when set, records events for *every* instantiate cache miss and every `reduce_conditional` decision, rendered as plain `trace: …` lines, not ariadne reports)

**Interfaces:**
- `newtype --trace-eval --input F.nt` prints to stderr, interleaved with harness output:
  - `trace: Name(arg, …) = <substituted body, pretty, one line>` per instantiate cache miss;
  - `trace: <check> <: <extends> → then|else|never|both` per conditional reduction.
- Flag defaults off; zero output and zero behavior change when off; composes with `dbg!`.
